# AgentsSync v1 产品规格

## 1. 概述

AgentsSync v1 是一个 SwiftUI 原生 macOS 主窗口应用，用于在同一台 Mac 上同步不同 AI 编程工具的 `skills` 目录，并通过可选 Git 共享库支持跨 Mac 同步。

首版只同步 `skills`，不处理 `commands`、`agents`、`rules`、MCP 配置或完整工具迁移。

## 2. 目标

- 让用户在 Codex、Claude Code、Cursor 和其他 AI 编程工具之间同步已安装或自定义的 skills。
- 在同步前清楚展示目录状态、差异和冲突，避免误覆盖用户内容。
- 在覆盖写入前保留本地备份，提供可恢复的安全兜底。
- 通过可选 Git 共享库支持跨 Mac 分发和同步 skills。

## 3. 非目标

- v1 不同步 `skills` 之外的目录。
- v1 不做后台实时监听或自动同步。
- v1 不自动解决冲突。
- v1 不内置 GitHub、GitLab 等平台登录。
- v1 不尝试转换不同工具之间不兼容的 skill 格式。
- v1 不实现菜单栏-only 应用形态。

## 4. 应用形态

v1 采用 macOS 主窗口应用形态。主窗口用于承载目录状态、差异列表、同步计划、冲突处理、历史记录和设置入口。

主窗口首页展示：

- 已连接的 skills 目录。
- 最近扫描时间。
- 差异数量。
- 冲突数量。
- “扫描差异”主操作。

## 5. 支持的目录类型

v1 支持以下目录来源：

- Codex 内置预设目录。
- Claude Code 内置预设目录。
- 用户手动添加的自定义 skills 目录。
- 用户手动配置的可选 Git 共享库目录。

Cursor 和其他工具在 v1 中通过自定义 skills 目录接入，不强行内置不确定的目录路径。

## 6. Skill 作用域

v1 必须区分 skill 目录的作用域。作用域不是普通标签，而是决定同步边界、覆盖关系和风险提示的核心规则。

AgentsSync 使用以下 3 级 skill 作用域：

- 全局级：工具、系统、插件或组织提供的基础 skills。全局级目录默认只读，不参与自动写入。
- 用户级：用户个人可复用的 skills。用户级是 v1 的默认主同步范围。
- 工程级：某个项目专属的 skills。工程级目录必须关联具体工程路径。

作用域优先级为：

```text
全局级 < 用户级 < 工程级
```

当不同作用域存在同名 skill 时，默认解释为高优先级作用域覆盖低优先级作用域，而不是普通冲突。例如，用户级和工程级同时存在 `writer`，应展示为“当前工程使用工程级版本”，而不是直接要求用户二选一覆盖。

## 7. 作用域同步规则

v1 的默认主同步范围是用户级 skills。Codex 和 Claude Code 的默认预设目录归为用户级。

工程级 skills 支持以下动作：

- 工程内同步：同一个工程内，不同 AI 编程工具的工程级 skills 可以互同步。
- 添加到当前工程：用户可将用户级 skill 复制到当前工程级目录。
- 提升为用户级：用户可将工程级 skill 复制到用户级目录。

工程级 skill 复制到用户级前，应用必须提示用户检查是否包含项目私有信息、业务上下文或敏感内容。

全局级 skills 支持以下动作：

- 扫描展示：应用可展示全局级目录中存在的 skills。
- 导入副本：用户可将全局级 skill 复制到用户级或工程级目录，形成可编辑副本。

全局级目录默认不允许写入。v1 不做全局级目录的双向同步。

跨作用域写入必须显式确认。用户级到用户级可以生成普通同步计划；用户级到工程级、工程级到用户级、全局级到其他作用域都必须以明确动作呈现，不允许静默同步。

## 8. 同步模型

v1 采用“目录互同步”模型。应用扫描多个已连接的 skills 目录，比较每个 skill 在各目录中的存在性与内容差异，并生成同步计划。

一个 skill 以 `skills` 目录下的一级子目录为同步单元。该目录内的 `SKILL.md`、`assets`、`references` 等文件作为整体复制、更新和备份。

同步流程：

1. 用户连接或确认 skills 目录。
2. 用户手动点击“扫描差异”。
3. 应用扫描所有已连接目录。
4. 应用生成差异列表和同步计划。
5. 用户处理冲突项。
6. 用户确认并执行同步。
7. 应用写入目标目录并记录结果。

## 9. 差异类型

扫描结果至少包含以下状态：

- 新增：某个 skill 只存在于部分目录，可复制到缺失目录。
- 缺失：某个目录缺少其他目录中已有的 skill。
- 内容不同：同名 skill 在多个目录中存在，但内容不一致。
- 无差异：同名 skill 在多个目录中内容一致，不需要写入。
- 冲突：同名 skill 在多个来源中都存在修改，不能安全自动决定保留哪一版。
- 作用域覆盖：同名 skill 存在于不同作用域，高优先级作用域版本覆盖低优先级作用域版本，不默认视为冲突。

## 10. 冲突处理

v1 不自动覆盖冲突项。

当同名 skill 在多个目录中内容不同且无法自动判断唯一来源时，应用将该项标记为冲突，并暂停该项同步。用户需要选择：

- 保留某个来源版本。
- 跳过该 skill。

用户完成冲突处理后，应用再生成最终同步计划。

只有同一作用域、同一同步组内的同名 skill 内容不同，才默认标记为冲突。跨作用域同名 skill 默认标记为作用域覆盖，并在 UI 中解释实际生效版本。

## 11. 写入与备份

执行任何覆盖写入前，应用必须在本地应用数据目录保存被覆盖 skill 的快照。

历史记录中应展示每次同步操作的结果，并提供从备份恢复上一版本的入口。

写入失败时，应用应展示错误信息，并停止依赖该失败步骤的后续写入。

## 12. Git 共享库

Git 是 v1 的可选跨设备能力，不影响本机目录互同步的基础功能。

用户可指定一个本地 Git 仓库作为 skills 共享库。应用调用本机 `git` 命令执行：

- `git status`
- `git pull`
- `git commit`
- `git push`

Git 认证交给系统 Git、SSH、Keychain 或用户已有凭据。v1 不在应用内管理 Git 平台账号和 OAuth 登录。

Git 共享库默认承载用户级 skills，可作为跨 Mac 分发用户级 skills 的桥梁。

工程级跨设备同步优先依赖工程自身的 Git 仓库，而不是 AgentsSync 的用户级 Git 共享库。v1 不默认把工程级 skills 推入个人 Git 共享库。

## 13. 设置

设置页用于管理：

- Codex 用户级预设目录。
- Claude Code 用户级预设目录。
- 自定义 skills 目录。
- 可选 Git 共享库路径。

用户添加自定义 skills 目录时，必须选择该目录的作用域。工程级目录必须记录对应工程路径。用户应能启用、禁用或移除已连接目录。

## 14. 测试场景

- 首次启动时能识别存在的 Codex 和 Claude Code skills 目录。
- 用户能添加自定义 skills 目录。
- Codex 和 Claude Code 默认预设目录被归类为用户级。
- 用户添加自定义目录时能选择全局级、用户级或工程级。
- 工程级目录必须记录对应工程路径。
- 单个 skill 只存在于一个目录时，扫描能生成复制到其他目标目录的计划。
- 两个目录存在同名且内容相同的 skill 时，不生成写入操作。
- 两个目录存在同名但内容不同的 skill 时，生成冲突并等待用户选择。
- 同名 skill 分别存在于用户级和工程级时，显示为作用域覆盖，不默认标记为冲突。
- 全局级 skill 默认只读，不生成写入全局级目录的同步操作。
- 用户级 skill 添加到工程级目录前需要明确确认。
- 工程级 skill 提升为用户级目录前需要提示检查项目私有信息。
- 执行覆盖更新前会创建本地备份。
- 用户可从历史记录恢复上一版本。
- Git 仓库未配置时，本机目录互同步可正常使用。
- Git 仓库已配置时，应用能展示 Git 状态。
- 用户确认后，应用能调用本机 Git 执行 pull、commit 和 push。
- Git 命令失败时，应用展示错误信息，不继续执行依赖该步骤的写入操作。

## 15. 默认假设

- v1 只面向 macOS。
- v1 使用 SwiftUI 原生实现。
- v1 只同步 `skills` 目录。
- v1 只支持手动同步。
- v1 以 skill 目录为同步单元，而不是逐文件同步。
- v1 默认主同步范围是用户级 skills。
- v1 默认不写入全局级 skills 目录。
- v1 默认不把工程级 skills 同步到用户级或 Git 共享库，除非用户明确选择“提升为用户级”。
- v1 的 Git 能力依赖用户本机已安装并可用的 `git` 命令。
