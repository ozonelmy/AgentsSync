# AgentsSync v1 产品规格

## 1. 概述

AgentsSync v1 是一个 SwiftUI 原生 macOS 主窗口应用，用于在同一台 Mac 上同步不同 AI 编程工具的 `skills` 目录，并通过可选 Git 共享库支持跨 Mac 同步。

首版只同步 `skills`，不处理 `commands`、`agents`、`rules`、MCP 配置或完整工具迁移。

## 2. 目标

- 让用户在 Codex、Claude Code 和其他 AI 编程工具之间同步已安装或自定义的 skills。
- 在同步前清楚展示目录状态、差异和冲突，避免误覆盖用户内容。
- 在覆盖写入前保留本地备份，提供可恢复的安全兜底。
- 通过可选 Git 共享库支持跨 Mac 分发和同步 skills。

## 3. 非目标

- v1 不同步 `skills` 之外的目录。
- v1 不做后台实时监听或自动同步。
- v1 不自动解决冲突。
- v1 不内置 GitHub、GitLab 等平台登录。
- v1 不尝试转换不同工具之间不兼容的 skill 格式。
- v1 不实现菜单栏-only 应用形态。

## 4. 应用形态

v1 采用 macOS 主窗口应用形态。主窗口用于承载目录状态、差异列表、同步计划、冲突处理、历史记录和设置入口。

主窗口首页展示：

- 已连接的 skills 目录。
- 最近扫描时间。
- 差异数量。
- 冲突数量。
- “扫描差异”主操作。

## 5. 支持的目录类型

v1 支持以下目录来源：

- Codex 内置预设目录。
- Claude Code 内置预设目录。
- 用户手动添加的自定义 skills 目录。
- 用户手动配置的可选 Git 共享库目录。

Cursor 和其他工具在 v1 中通过自定义 skills 目录接入，不强行内置不确定的目录路径。

## 6. 同步模型

v1 采用“目录互同步”模型。应用扫描多个已连接的 skills 目录，比较每个 skill 在各目录中的存在性与内容差异，并生成同步计划。

一个 skill 以 `skills` 目录下的一级子目录为同步单元。该目录内的 `SKILL.md`、`assets`、`references` 等文件作为整体复制、更新和备份。

同步流程：

1. 用户连接或确认 skills 目录。
2. 用户手动点击“扫描差异”。
3. 应用扫描所有已连接目录。
4. 应用生成差异列表和同步计划。
5. 用户处理冲突项。
6. 用户确认并执行同步。
7. 应用写入目标目录并记录结果。

## 7. 差异类型

扫描结果至少包含以下状态：

- 新增：某个 skill 只存在于部分目录，可复制到缺失目录。
- 缺失：某个目录缺少其他目录中已有的 skill。
- 内容不同：同名 skill 在多个目录中存在，但内容不一致。
- 无差异：同名 skill 在多个目录中内容一致，不需要写入。
- 冲突：同名 skill 在多个来源中都存在修改，不能安全自动决定保留哪一版。

## 8. 冲突处理

v1 不自动覆盖冲突项。

当同名 skill 在多个目录中内容不同且无法自动判断唯一来源时，应用将该项标记为冲突，并暂停该项同步。用户需要选择：

- 保留某个来源版本。
- 跳过该 skill。

用户完成冲突处理后，应用再生成最终同步计划。

## 9. 写入与备份

执行任何覆盖写入前，应用必须在本地应用数据目录保存被覆盖 skill 的快照。

历史记录中应展示每次同步操作的结果，并提供从备份恢复上一版本的入口。

写入失败时，应用应展示错误信息，并停止依赖该失败步骤的后续写入。

## 10. Git 共享库

Git 是 v1 的可选跨设备能力，不影响本机目录互同步的基础功能。

用户可指定一个本地 Git 仓库作为 skills 共享库。应用调用本机 `git` 命令执行：

- `git status`
- `git pull`
- `git commit`
- `git push`

Git 认证交给系统 Git、SSH、Keychain 或用户已有凭据。v1 不在应用内管理 Git 平台账号和 OAuth 登录。

Git 共享库与其他 skills 目录一样参与扫描和同步计划，可作为跨 Mac 分发 skills 的桥梁。

## 11. 设置

设置页用于管理：

- Codex 预设目录。
- Claude Code 预设目录。
- 自定义 skills 目录。
- 可选 Git 共享库路径。

用户应能启用、禁用或移除已连接目录。

## 12. 测试场景

- 首次启动时能识别存在的 Codex 和 Claude Code skills 目录。
- 用户能添加自定义 skills 目录。
- 单个 skill 只存在于一个目录时，扫描能生成复制到其他目标目录的计划。
- 两个目录存在同名且内容相同的 skill 时，不生成写入操作。
- 两个目录存在同名但内容不同的 skill 时，生成冲突并等待用户选择。
- 执行覆盖更新前会创建本地备份。
- 用户可从历史记录恢复上一版本。
- Git 仓库未配置时，本机目录互同步可正常使用。
- Git 仓库已配置时，应用能展示 Git 状态。
- 用户确认后，应用能调用本机 Git 执行 pull、commit 和 push。
- Git 命令失败时，应用展示错误信息，不继续执行依赖该步骤的写入操作。

## 13. 默认假设

- v1 只面向 macOS。
- v1 使用 SwiftUI 原生实现。
- v1 只同步 `skills` 目录。
- v1 只支持手动同步。
- v1 以 skill 目录为同步单元，而不是逐文件同步。
- v1 的 Git 能力依赖用户本机已安装并可用的 `git` 命令。
